Note2: This flow illustrates fitting one model. If the goal is to fit multiple models, the user needs to implement a loop to pass each model in turn. I will add a demo for this particular usecase during implementation

Notes
  • I've mostly added public functions that researchers can call directly: Chances are that I will need to implement some private helper functions along the way. Will add those to this doc as I go
  • We are currently working on moving away from local .csv files to a central data repository, hence the load_data_from_db() function
  • As discussed in the GDoc: I have ommited a collection of cleaning functions for now. Instead, I assume that users use the template in some capacity and either load a precleaned data set or frankenstein cleaning routines between the load_data() and summarise() steps
  • Ideally, summarise() would be parametrised, and users pass things like column names for whatever variables they want summary stats. Additionally, they can toggle plots. The plotting functions themselves are private helpers (can't be called directly)
  • The familty in the fit_reg() function will be a parameter
